Guide

Step-by-step workflow for processing Sea Duty Cert sheets, fixing validations, and rebuilding clean outputs.

1) Before you click Process

  1. Sea Duty Cert (PDF): Select the certification sheet PDFs (you can upload multiple).
  2. Rate CSV (optional but recommended): Upload the rates file if you need updated pay logic for the run.
  3. Template PDF (optional): Only upload if you want to override the default NAVPERS 1070/613 template used by the container.
  4. Strikeout color: Pick Black or Red. This controls the color used on strikeout PDFs for invalid/skipped items.

2) Run a batch

  1. Click ‚ñ∂Ô∏è Process and watch Status + the Live Processing Log.
  2. Do not start another run until Status shows COMPLETE (or ERROR).
  3. If you uploaded the Rate CSV, the backend reloads rates for this run automatically.

3) Review, override, and keep your counts honest

  1. In Downloads & Member, pick a Member, then a Sheet.
  2. Use the tables under Review & Override:
    • Override = force Valid, force Invalid, or leave Auto.
    • Reason = type why you changed it (helps later when you audit your work).
  3. When you make edits, the UI tracks them as ‚Äúpending‚Äù until you click üíæ Save Overrides.
  4. After saving, you can reload the sheet and your override should still be there.
Golden rule: Save overrides first, then rebuild outputs. If you rebuild without saving, you are rebuilding old data.
Counts: The ‚úÖ and ‚ùå tiles reflect what the UI currently shows for the selected sheet. If you switch sheets, counts change with it.

4) Rebuild clean outputs

  1. Click üîÑ Rebuild Outputs to regenerate PDFs/summary using your saved overrides.
  2. Use üì¶ Package for a single zip bundle, or üìÇ All for the full output set.
